The problem

AI video tools spit out filenames like
Gen-3 Alpha Turbo 1019957322, zoom out, pan out, l, 00290-3597567898 cop, M 5_prob4.mp4

When you generate dozens of videos from source images using tools like Runway, Pika, or Luma, the resulting filenames are unsortable garbage. Manually renaming each one to something like saloon-night-shootout.mp4 takes hours. ZohoVideoRenamer does it in minutes.

Two modes

Pick the workflow that fits your files.

Same review UI, same bulk-rename pipeline, same undo log. The only difference is what the AI looks at to propose names.

Mode 1

Match mode

You have both a folder of source stills and a folder of generated videos. The tool matches each video to its source still by filename substring, then names everything based on the still's content.

When to use it: Output from AI video generators like Runway / Pika / Luma where the source-still filename is embedded in the video filename. Typical match rate is 90 %+ on the first try.

Mode 2 · new in v0.1.8

Catalog mode

You have only videos — random-named clips from a phone, drone, screen capture, etc. The tool extracts three frames per video (start / middle / end) and the AI looks at all three to propose one descriptive 3-word name per clip.

When to use it: Folders of IMG_0234.mp4, DJI_0089.mp4, GH010012.mp4 with no source stills. Toggle the "Videos only" checkbox in the app and pick just the videos folder.

Catalog mode — the review UI

Random-named videos in the sidebar (IMG_0234.mp4, MOV_4471.mp4, …), AI-proposed names in the form field, click-to-exclude on each card, identical review flow to Match mode.

Three ways to apply the renames

When you commit, pick how the tool touches your files:

Rename in place
Default. Originals stay where they are, just with new names. Reversible via undo log.
Copy to new folder
Safest. Originals untouched; renamed copies land in the output folder.
Move to new folder
Cleans up the source folder. Works across filesystems unlike pure rename.
How it works

From folder of garbage filenames to clean library — in one window.

No subscriptions, no cloud uploads, no lock-in. Everything runs locally except the (optional) AI naming calls, which use your own API key. No browser, no terminal, no JSON file wrangling.

01

Open the app, pick folders

Pick the folder with your videos (and stills, if you have them). Paste your Anthropic or OpenAI key if you want AI-generated descriptive names. Hit Start.

02

AI proposes names

The app extracts frames from each video and an AI vision model proposes a 3-word descriptive name — mountain-sunset-vista, desert-cactus-dawn. You see them pre-filled in the next screen.

03

Review in the same window

The window switches to the review screen. Each entry shows the still next to its matched videos. Hit Enter to accept the AI name, edit any that need fixing, click-toggle to exclude wrong matches.

04

Click "Approve & Rename All"

One button. Files are renamed in place (or copied / moved to a new folder, your choice). A timestamped undo log lets you reverse the whole run if something looks off.

Review UI

A web interface that respects your time.

Built to chew through hundreds of entries quickly. Keyboard-driven (Enter approves, S skips, F flags), live preview of exactly which files will move where, side-by-side still vs. video frame thumbnails, and a play button on every video card so you can verify a match without leaving the page.

State is held in browser localStorage, so you can close the tab, take a break, and pick up where you left off.

No surprises

See every rename before it happens.

The name input shows a live preview of every file that would be moved. Type a name; the preview updates as you type. Multiple videos for the same still automatically get -v1, -v2, -v3 suffixes. Original and "copy" variants of the same still get distinct destinations so nothing overwrites.

When you actually apply, an undo log lands in the project folder. One command reverses the entire run.

⚠ First launch — you'll see a security warning

The app is not code-signed (Apple Developer ID and Microsoft EV cert cost hundreds per year, and this is a free OSS tool). That means your OS will refuse to open it on first launch. Here's how to allow it — takes 10 seconds. Subsequent launches work normally.

🍎 macOS

When you see "ZohoVideoRenamer can't be opened because Apple cannot check it for malicious software":

  1. Click Done on the warning
  2. Open System Settings → Privacy & Security
  3. Scroll down to the Security section — you'll see "ZohoVideoRenamer.app was blocked to protect your Mac"
  4. Click Open Anyway on the right (see arrow →)
  5. Re-launch the app — it'll open normally from now on

Faster alternative: In Finder, right-click the app → Open → Open on the confirmation dialog. Done.

Power-user one-liner: xattr -dr com.apple.quarantine /Applications/ZohoVideoRenamer.app

🪟 Windows

When you see the blue "Windows protected your PC" SmartScreen dialog:

  1. Click More info (small link in the dialog)
  2. Click the Run anyway button that appears
  3. The app opens — Windows will remember your decision

Or: Right-click the .exe → Properties → check Unblock at the bottom → OK → double-click as normal.

These warnings appear because the binary isn't signed by Apple/Microsoft, not because anything is wrong with the code. The source is fully auditable at github.com/princezoho/ZohoVideoRenamer.

For power users · scripts & CI

Same flow available as a CLI.

Most people should just use the desktop app — everything happens in one window. But if you want to script this into a pipeline, integrate with CI, or batch-process many folders, the same engine ships as a CLI.

# 1. Walk both folders, match videos to stills, generate thumbnails
zoho-video-renamer scan \
  -s ~/Pictures/source-stills \
  -v ~/Movies/generated-videos \
  -o ./review

# 2. (Optional) Ask an AI to propose 3-word names
export ANTHROPIC_API_KEY=sk-ant-...
zoho-video-renamer ai-name -o ./review --provider anthropic

# 3. Open the review UI in your browser
zoho-video-renamer review -o ./review

# 4. Dry-run, then execute. An undo log is written automatically.
zoho-video-renamer apply -o ./review -a ~/Downloads/rename-approvals.json
zoho-video-renamer apply -o ./review -a ~/Downloads/rename-approvals.json --execute

# 5. If anything looks wrong, undo:
zoho-video-renamer undo --log ./review/rename-undo-*.json --execute
FAQ

Common questions.

Does it send my files anywhere?
No. Files never leave your machine. The only network traffic is the (optional) vision API calls for name proposals, which go from your machine directly to your chosen AI provider using your own API key. Skip the AI step and there's zero network activity.
Which AI providers are supported?
Anthropic Claude (Sonnet 4.5 by default) and OpenAI GPT-4o. You bring your own API key from console.anthropic.com or platform.openai.com. The model is configurable if you want to use a cheaper / faster variant.
How does the matching actually work?
It looks at each video filename and tries to extract a "stub" — either a numeric ID pattern like 00290-3597567898 (common in AI image generators), or a manual label like bg14, title-blah, or any literal substring that matches one of your stills' filenames. So if your still is mountain-sunset.png and your video is pika-render-mountain-sunset-v2.mp4, the substring mountain-sunset matches.
What if some videos don't match anything?
They're listed in unmatched-videos.json and left alone. You can match them manually after the fact, or rerun with different stills folders. Nothing gets moved or renamed without your approval.
What if I want to undo?
Every rename writes a timestamped rename-undo-*.json log to your output folder. Power users can run zoho-video-renamer undo --log path/to/rename-undo-*.json --execute from a terminal to reverse every move. (A one-click Undo button inside the app is on the roadmap.) The log records each rename as (current path → original path), so even partial-failure runs are reversible.
Does it work on Linux?
The Python package runs anywhere Python 3.9+ runs. The desktop app uses pywebview which supports macOS (WKWebView), Windows (WebView2), and Linux (GTK). Pre-built downloads ship for macOS and Windows; Linux users can pip install zoho-video-renamer[all] and run python -m zoho_video_renamer.embedded to launch the same single-window app.
Do I have to leave the app to use it?
No. Since v0.2.0 everything happens inside one native window — the setup form, the AI naming step, the review of matched videos, and the final rename are all one continuous flow. No browser tabs, no JSON files to export, no terminal commands. The old multi-step browser+CLI flow still works via the CLI but the desktop app no longer needs any of that.
Will it overwrite my files?
No. Every rename is checked for collisions before execution. If two source files would end up at the same target path, the run aborts with a clear error rather than overwriting. The review screen shows you exactly what each file will be renamed to before you ever commit. You can also pick "Copy to new folder" mode which leaves your originals completely untouched.
Can I contribute?
Please do. The codebase is small (~1500 lines of Python). PRs and issues welcome on GitHub. See docs/architecture.md for a tour.
